# RetailFlow Dashboard — Roadmap

Roadmap público do projeto RetailFlow Dashboard.

O objetivo deste roadmap é mostrar a evolução planejada da demo front-end para uma aplicação SaaS mais realista, com autenticação, banco de dados e fluxos comerciais mais completos.

---

## Fase 1 — Demo Front-end Atual

Status: em andamento

- Dashboard responsivo
- CRUD de clientes
- CRUD de pedidos
- CRUD de pagamentos
- Métricas financeiras
- Gráficos e relatórios
- Persistência local com localStorage
- Deploy na Vercel

---

## Fase 2 — Melhorias de UX e Produto

- Página individual de cliente
- Histórico de pedidos por cliente
- Histórico financeiro por cliente
- Empty states mais profissionais
- Loading states
- Error states
- Melhorias de responsividade mobile
- Melhorias de acessibilidade
- Melhor organização de filtros

---

## Fase 3 — Validação e Qualidade

- React Hook Form
- Zod para validação
- Testes básicos de componentes
- Testes de fluxos principais
- Melhor tratamento de erros
- Melhor padronização de componentes

---

## Fase 4 — Backend Real

- Integração com Supabase
- Banco de dados PostgreSQL
- Autenticação de usuários
- Rotas privadas
- Persistência real de clientes, pedidos e pagamentos
- Regras de segurança com RLS

---

## Fase 5 — Funcionalidades SaaS

- Multiusuário
- Controle por empresa/loja
- Exportação de relatórios
- Notificações financeiras
- Dashboard por período
- Gestão avançada de inadimplência
- Configurações da conta

---

## Visão de Produto

O RetailFlow tem como objetivo evoluir de uma demo de portfólio para uma base de produto SaaS simples voltado para pequenos negócios que precisam acompanhar clientes, pedidos e pagamentos de forma visual e organizada.
